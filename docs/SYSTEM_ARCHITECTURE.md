```
┌─────────┐    POST /api/inventory/transaction    ┌─────────┐    Validate    ┌─────────┐

└─────────┘    {location_id, item_id, date,      └─────────┘    Schema       └────┬────┘
                 received, issued}                                             │
                                                                                │
                                                                                ▼
┌─────────┐    Response    ┌─────────────┐    Business    ┌──────────────┐    ┌─────────┐

```

## 4. Technology Stack Summary

| Layer | Technology | Purpose |
|-------|------------|---------|
| **Frontend** | React + Vite + Tailwind | UI Framework |
| **Backend** | FastAPI | REST API Server |
| **ORM** | SQLAlchemy | Database Abstraction |
| **Primary DB** | SQLite (dev) / PostgreSQL (prod) | Transactional Data |
| **Vector DB** | ChromaDB | AI Semantic Memory |
| **AI Framework** | LangGraph + LangChain | Agent Orchestration |
| **LLM** | Groq (OpenAI-compatible) | Language Model |
| **Speech-to-Text** | Sarvam AI | Voice Input |
| **Observability** | LangSmith (optional) | Tracing |

## 6. Module Responsibilities

| Module | Responsibility | Public API |
|--------|---------------|------------|
| `api/routes` | HTTP endpoint definitions | REST endpoints |
| `services/*_service` | Business logic | Domain operations |
| `repositories/*_repo` | Data access | CRUD operations |
| `services/ai_agent` | AI conversation | `agent.query()` |
| `services/memory` | Vector storage | `add_message()`, `search_relevant()` |
| `database` | ORM models, connections | Session, models |
| `core` | Config, exceptions, logging | Settings, error handling |
